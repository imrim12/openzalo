# Research & Analysis Phase

**When to skip:** If provided with researcher reports, skip this phase.

## Core Activities

### 1. Codebase Investigation
- Explore the project structure and existing patterns.
- Search for specific implementations or usage of libraries (e.g., how Drizzle is used, how Reka UI components are composed).
- Consult the `README.md` and database schema for the foundational architecture.

### 2. External Research
- Search for and read official documentation for technologies in the stack (Nuxt, Drizzle, Reka UI, etc.).
- Fetch specific documentation pages or technical articles when needed.


## Best Practices

- Research breadth before depth.
- Document findings for the Synthesis phase.
- Identify multiple approaches for comparison.
- Consider edge cases and security early.
- Focus on how new research integrates with the existing architecture.
